1 PocketBase in Coolify deployen

1
Neuen Service anlegen Coolify Dashboard → New Resource → Docker Image
2
Image konfigurieren 
Image:    ghcr.io/muchobien/pocketbase:latest
Port:     8090
3
Persistentes Volume anlegen 
Source:       pb_data   (persistent volume)
Destination:  /pb/pb_data
Ohne dieses Volume gehen alle Daten beim Neustart verloren!
4
Domain zuweisen Z.B. kw.hofreither.at — Coolify übernimmt SSL automatisch (Let's Encrypt).
5
Deployen Auf Deploy klicken. Nach ~1 Minute ist PocketBase unter
https://kw.hofreither.at/_/ erreichbar.

2 PocketBase Admin einrichten

1
Admin-Account erstellen https://kw.hofreither.at/_/ aufrufen → Admin-E-Mail und Passwort vergeben.
2
Collection kw_state erstellen Collections → New collection
NameTypPflicht
dataJSONnein

API-Rules (alle 4 Felder):

# Nur eingeloggte Nutzer dürfen lesen/schreiben:
@request.auth.id != ""
3
Benutzer anlegen Users → New record — für jede Dame einen Account:

NameE-MailPasswort
Dame 1dame1@hofreither.at(sicher wählen)

3 Frontend in PocketBase ablegen

Das Frontend wird als index.html direkt im PocketBase-Container abgelegt. So läuft alles unter einer einzigen Domain — kein zweiter Service nötig.

1
Standalone-HTML vorbereiten Die Datei Kühlwagen-Verwaltung-standalone.html zu index.html umbenennen.
2
Datei in den Container kopieren Über Coolify → Terminal des laufenden Containers, oder via SSH + Docker: 
# Container-ID herausfinden:
docker ps | grep pocketbase

# HTML in pb_public kopieren:
docker cp index.html <container-id>:/pb/pb_public/index.html
Alternativ: In Coolify unter File Manager direkt hochladen, falls verfügbar.
3
Fertig! App aufrufen: https://kw.hofreither.at/index.html
Login-Maske erscheint → mit den angelegten Benutzer-Accounts anmelden.

4 Updates einspielen

Wenn du eine neue Version der App entwickelt hast:

  1. In diesem Projekt standalone HTML neu generieren (super_inline_html)
  2. Zu index.html umbenennen
  3. Per docker cp in den Container kopieren (Schritt 3.2)
  4. Fertig — die Damen sehen die neue Version beim nächsten Browser-Reload
Optional: GitHub Actions kann den Copy-Schritt automatisieren — dann reicht ein git push.

5 Datensicherung

Automatisches Backup (empfohlen)

PocketBase hat ein eingebautes Backup-System:

# In pb_hooks oder via Admin UI:
Admin → Settings → Backups → Schedule: "täglich 03:00"

Manuelles Backup

Das gesamte Datenvolume (/pb/pb_data) enthält die SQLite-Datenbank. Einfach kopieren:

docker cp <container-id>:/pb/pb_data ./backup-$(date +%Y%m%d)

App-internes Backup

Die Damen können jederzeit über Stammdaten → Backup exportieren eine JSON-Datei herunterladen und lokal aufbewahren.

7 E-Mail-Benachrichtigungen einrichten

SMTP-Anbieter: Brevo (empfohlen)

Brevo ist kostenlos (300 Mails/Tag), braucht keine Server-Installation — nur Zugangsdaten konfigurieren.

1
Account erstellen
Kostenlos registrieren auf brevo.com → E-Mail-Adresse bestätigen
2
SMTP-Zugangsdaten holen
Brevo Dashboard → SMTP & API → Tab SMTP → Daten notieren:
Host, Port (587), Login, Passwort
3
In PocketBase konfigurieren
kw.hofreither.at/_/ → Settings → Mail settings:
FeldWert
SMTP Hostsmtp-relay.brevo.com
SMTP Port587
UsernameDeine Brevo-E-Mail
PasswordBrevo SMTP-Passwort
Sender Addressbuchung@hofreither.at
Sender NameKühlwagen-Verleih Hofreither
4
Hook-Datei in den Container kopieren 
# Datei pb_hooks/kw_anfragen.pb.js auf Server hochladen:
docker cp kw_anfragen.pb.js <container-id>:/pb/pb_hooks/kw_anfragen.pb.js

# Container neu starten damit Hooks geladen werden:
docker restart <container-id>
Die Hook-Datei liegt im Projekt unter pb_hooks/kw_anfragen.pb.js
